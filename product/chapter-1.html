Implementing a Static Blog System

A simple and easy tutorial for beginners

Starting to Make a Template

Since you've already have some knowledge about HTML, here let's build a simplest HTML document and name it index.html. It looks like this:


<!DOCTYPE html>
<html>
<head>
	<meta charset="utf-8" /> <!-- Specify edcoding -->
	<title>My Blog</title> <!-- Document title -->

	<!-- Following: All stylesheets we are going to include -->
	<link rel="stylesheet" type="text/css" href="css/reset.css">
	<link rel="stylesheet" type="text/css" href="css/theme_common.css">
	<link rel="stylesheet" type="text/css" href="css/theme_desktop.css">
	<link rel="stylesheet" type="text/css" href="css/theme_mobile.css">
	<link rel="stylesheet" type="text/css" href="css/personal.css">

	<!-- Following: All scripts we are going to run -->
	<script type="text/javascript" src="js/func.js"></script>
</head>
<body>
</body>
</html>

Here, we have added some tags in <head>, including some stylesheets and a script, which can be worked out later. As you have already know what do they mean, I won't repeat the uses of them. If you open this file in your browser, it's empty.

Let's move on. As a blog homepage, we need some blocks playing different roles in the page. For example, a header for website titls and navigations, a footer for copoyright note and links to friends' blogs, and finally an area for your posts. On the design level, we are going to show ten most recent posts directly as full text, and older posts will be displayed as a link on a list. You may ask, "What if I have at most ten posts in total?" Good question, but leave it later to be solved. Algorithm-related issues are not this series' topic.

So let's add a <header> and a heading (<h1>) inside the <header>, and corresponding <footer> and the copyright note. Don't forget a container for the posts. So finally it looks like this:


<!DOCTYPE html>
<html>
<head>

	<meta charset="utf-8" /> <!-- Specify edcoding -->
	<title>My Blog</title> <!-- Document title -->

	<!-- Following: All stylesheets we are going to include -->
	<link rel="stylesheet" type="text/css" href="css/reset.css">
	<link rel="stylesheet" type="text/css" href="css/theme_common.css">
	<link rel="stylesheet" type="text/css" href="css/theme_desktop.css">
	<link rel="stylesheet" type="text/css" href="css/theme_mobile.css">
	<link rel="stylesheet" type="text/css" href="css/personal.css">

	<!-- Following: All scripts we are going to run -->
	<script type="text/javascript" src="js/func.js"></script>

</head>
<body>

	<!-- Here's the header -->
	<header class="global" id="global-header">
		<h1>My Blog</h1>
	</header>

	<!-- Here's the container -->
	<div class="cont" id="cont">
	</div>

	<!-- Here's the footer -->
	<footer class="global" id="global-footer">
		Copyright © John Appleseed. All Rights Reserved.
	</footer>

	<!-- Here's the JavaScript -->
	<script type="text/javascript" src="js/main.js"></script>

</body>
</html>

Have you noticed those attributes on the tags? Yeah, they are useful to specify an element's styles, and they will be useful to locate an element with JavaScript. In JavaScript, there's a built-in function named document.getElementById(), which allows you get the header in the document with document.getElementById("global-header") and then you can access the element. We'll talk about this in next chapter.

We say "tags" in HTML source code, and say "elements" in rendered webpages. Before you dig deeper, it's okay to think rendered the relation between webpages and HTML source code is same to that between binary executable files and C/C++ source code.

As you have seen, a <script> can be added everywhere in either <head> of <body>. You could just consider where it is referenced, where it will run at. For example, if you run some script before <footer> using document.getElementById("global-footer") to access the footer, you'll get an error saying null is not an object.

Here we have almost done the HTML template. In next chapter, we'll go start writing JavaScript.
